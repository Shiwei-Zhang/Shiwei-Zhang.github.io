4.git中的三个区

工作流

  • 工作区
    通过git add 添加到暂存区

  • 暂存区
    特点:过渡的作用，避免误操作，保护工作区和历史区，分支处理;
    为了保护工作区和历史区，过渡作用
    通过git commit 添加到历史区

  • 历史区
    查看历史状态

修改时通过git status查看当前状态

7.撤销

  • 从缓存区中撤销add内容

  • 撤回文件
    如果工作区有更改 ，我想用暂存区域或者历史区的内容去覆盖掉我们的工作区域
    先查找暂存区 ，查找不到 查找历史区
    用暂存区 替换掉工作区里的内容

  • 合并提交
    如果我们使用git commit 会产生一个新的版本，但是我们此次的提交，还想算作上次的提交

如果进入编辑模式按esc键 输入:wq

8.删除

  • 删除缓存区，但是要保证工作区里内容必须不存在

  • 只删除缓存区

  • 同时删除暂存区和工作区

9.恢复

  • 恢复某个版本文件

  • 从版本库拉回代码 回滚 回到过去
    git log 查看版本号

  • 在回到未来
    通过git reflog 查看未来的版本，使用reset回到未来
